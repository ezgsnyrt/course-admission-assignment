    Rock-Paper-Scissors Game
    (Individual Project)

    Project Description and Code Explanation

    This projects aims to create a rock, paper, scissors game using HTML, CSS and JavaScript together with React. I started developing the game by creating the React project. After that, I wrote the basic HTML and CSS codes which enabled me to prepare the initial structure of the game. I also helped me monitor and follow the changes simultaneously. Then, I integrated JavaScript into my existing codes.

    This project basically has two components. The first component is the App.js which involves all the variable definitions, functions and HTML contents. This compont has an ‘actions’ object which defines the rules of the game and indicates which action beats which one. Also, there is a function for random actions which select actions randomly from the actions object. Herein, there is another function which compares the player’s and the computer’s actions and reflect the result on the score board. Another function included in this component is ActionIcon which renders the proper icon based on the given action. I’ve also included another function which displays the player’s or the computer’s score and their current action. In order for the player to take an action, the game includes a button action function which lets the player select an action. When clicked on the action button, it calls another function with the selected action. Moreover, this component includes a function which calls a text about the result of each turn: whether it’s a tie, the player wins, or the computer wins. Besides, the App function is the backbone of the game which contains the game logic. It maintains the state for the player’s and computer’s actions, their scores, and the game result basically. It also helps handle the player’s action, generates a random action for the computer and calculates the winner, and updates the score board. It also includes the reset button which enables the player to reset the player’s and computer’s score when clicked on this button.

    The second main component is the Modal created for allowing the players learn the game rules. This component includes a state variable to control when the modal will be visible. For instance, the state variable ‘modal’ is set to false, resulting in not showing the modal at the start. Corresponsingly, in the modal, there is also a ‘game rules’ button which triggers a toggle function to change the visibilty of the modal. When the player clicks on the button, modal functions becomes true. Consequently, a modal window with its content is rendered. On the other hand, when the player clicks on the ‘close’ button inside the modal, the toggle modal function is called again and the modal disappears.

    In addition to the aforementioned features, I incorporated media queries into the project in order to make it responsive in terms of design. I mean it is expected to be played in small, medium and large sized devices with the help of the adaptive design of the game content.
